The format

Markdown tells you how it looks.
RAOS tells your AI what it is.

One file. Human-readable on the left, machine-addressable on the right. Typed blocks like task, memento, and chapter replace ad-hoc conventions. Queries return precise, typed results — not fuzzy grep matches.

notes.raos RAOS 
@raos/1.0
@type: project.overview
@title: Product Launch Plan
@tags: launch, product, q2

:: identity
  %% project {
    goal: "first 100 users"
    audience: "technical, skeptical"
  }

:: context
  %% memento layer=2 {
    "Tone: honest, concrete."
    "Zero marketing speak."
  }

:: tasks
  %% task {
    title: "Ship landing page"
    priority: "high"
    due: "2026-05-01"
  }
raos search CLI 
# Find every high-priority task
$ raos search --block task \
    --field priority --value high

# Query by date, type, relation
$ raos search --due-before 2026-06-01 \
    --type "project.*"

# Semantic search, fully local
$ raos rag index ~/notes/
$ raos rag search "launch deadlines"

# Pipe structured results anywhere
$ raos search --block task --json \
    | jq '.[] | .title'

What's inside

Built for knowledge work that thinks alongside AI — not against it.

Start simple. Grow deep.

Write one line or a full schema. Four progressive layers let you adopt structure at your own pace — no upfront commitment, no migration tax.

Context your model remembers

The memento layer gives AI persistent instructions, voice, and framing per file. Your models stay on-brand without re-prompting every time.

Queries that mean something

Search by block type, relation, date range, or priority. Over twenty structured filters turn your notes into something you can actually query — not just scroll.

Private semantic search

Local RAG with Ollama and FAISS ships built-in. Your embeddings never leave your machine. No API keys, no monthly bills, no data leaving your desk.

Markdown stays readable

RAOS parses .raos and .md with the same engine. Your existing notes don't need migration — open, query, and extend them with structure the moment you're ready.

Git-friendly from day one

Plain text means perfect diffs. Review your knowledge base like code. Branch your notes for experiments. Merge them back when ideas prove themselves.

Who it's for

Three kinds of work RAOS makes dramatically easier.

Knowledge workers

Stop rebuilding the same context in every new chat.

You're working out of Obsidian, Notion, or a folder of markdown files. Every new AI session starts with "let me explain my project again". RAOS encodes your context once, queries it forever.

Researchers & writers

Ground your AI in your actual sources, not its hallucinations.

You have hundreds of notes on papers, authors, and findings. Typed metadata turns that mess into a real research base — one your local RAG can answer from, with citations that point to files you wrote.

Builders & operators

Give your agent fleet a file format they can actually parse.

Running multiple local LLMs that hand off tasks to each other? RAOS plus the wire SDK gives them a shared, deterministic language — no more duct-taping JSON schemas between ollama calls.

Design principles

The things we refuse to compromise on.

Principle · 01
Your files belong to you
Everything is plain text. Everything runs on your machine. No account required, no cloud dependency, no telemetry. If RAOS disappeared tomorrow, your files would still open in any editor.
Principle · 02
Model-agnostic by design
RAOS doesn't care which LLM you use. Local Ollama, Claude, GPT-4, whatever comes next — the format stays the same. Switch models anytime without rewriting a single note.
Principle · 03
Open standard, open code
The specification is CC-BY-SA. The reference implementation is Apache 2.0. Fork it, extend it, embed it in your product. No API limits, no enterprise tier, no surprise pricing page.
